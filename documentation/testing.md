# UI/API тестирование текущего workflow

Дата актуализации: 22.04.2026.

## Окружение

- Frontend: `http://localhost:5173`
- Backend: `http://127.0.0.1:8080`
- Тестовые пользователи:
  - `admin1` / `1234` — ADMIN
  - `initiator1` / `1234` — INITIATOR
  - `feo1` / `1234` — FEO
  - `cashier1` / `1234` — CASHIER
  - `accountant1` / `1234` — ACCOUNTING
  - `director1` / `1234` — DIRECTOR

## Общие правила теста

- UI является основным каналом проверки.
- API используется для контроля статусов, прав, истории, файлов и уведомлений.
- Тестовые заявки оставляются в базе с уникальными маркерами.
- Все действия проверяются через новый слой workflow-действий: главное действие в строке и меню `...`.
- Колонка **«Согласование»** должна показывать только тег статуса, без управляющего Select.

## 11. Регрессия после изменений 22.04.2026

Обязательно проверять:

- в UI используется термин «исключение из регламента»;
- `PENDING_GATE` отображается как «Требует исключения»;
- `SUSPENDED` отображается как «Отложена»;
- нет inline Select в колонке «Согласование»;
- preview в форме совпадает с фактическим результатом `submit`;
- backend endpoint `POST /requests/gate_preview` учитывает матрицу ДДС;
- `cashier1` не может отложить заявку;
- `feo1` может отложить и вернуть заявку на согласование.

## 12. Автоматический regression suite

Запускать из папки `frontend`.

Команды:

- `npm run build` — проверка TypeScript и production-сборки;
- `npm run test:api` — API-регрессия workflow и `gate_preview`;
- `npm run test:e2e` — UI-регрессия Playwright;
- `npm run test:e2e -- --headed` — UI-регрессия в видимом браузере;
- `npm run test:regression` — полный набор API + UI;
- `npm run test:regression:headed` — полный набор в видимом браузере.

Требования к окружению:

- frontend доступен на `http://localhost:5173`;
- backend доступен на `http://127.0.0.1:8080/api/v1`;
- тестовые пользователи доступны с паролем `1234`;
- НСИ и календари заполнены.

Текущий результат прогона 22.04.2026:

- `npm run build` — PASS;
- `npm run test:api` — PASS, 6/6;

## 13. Рабочее пространство казначея

Проверки:

- `cashier1` и `feo1` видят пункт меню `Рабочее пространство казначея` по праву `cashier_workspace_view`;
- при открытии `/cashier` дата оплаты по умолчанию равна текущему дню;
- в экран попадают только `APPROVED + UNPAID` и `PAID`;
- вкладки строятся только по организациям, где есть заявки в текущем фильтре;
- внутри вкладки таблица плоская, без группировок;
- `cashier1` может оплатить `APPROVED + UNPAID`, `feo1` видит экран без действия оплаты;
- `PAID` нельзя оплатить повторно;
- настройка колонок применяется ко всем вкладкам рабочего пространства;
- `Выгрузить Excel` видна только при праве `req_export_excel` или superadmin;
- экспорт выгружает текущий вид: текущий фильтр, текущую вкладку, видимые колонки и их порядок.
