### Б-15 — Повторная оплата не блокируется (ИСПРАВЛЕН)

| Поле | Значение |
|------|---------|
| Обнаружен | 17.04.2026, API-тестирование |
| Статус | ✅ Исправлен |
| Серьёзность | Высокий |

**Воспроизведение:** `POST /api/v1/requests/{id}/pay` повторно для уже оплаченной заявки (payment_status = PAID) — возвращал 200 OK вместо 400.

**Причина:** `update_payment_status` в `request_service.py` не проверял текущий статус оплаты. Также отсутствовала проверка что заявка APPROVED.

**Исправление:** В `app/services/request_service.py` добавлены проверки перед сменой статуса:
- Если `payment_status == PAID` → 400 «Заявка уже оплачена»
- Если `approval_status != APPROVED` → 400 «Оплатить можно только утверждённую заявку»
