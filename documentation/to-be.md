# Целевое состояние процесса

Дата актуализации: 22.04.2026.

## Цель

Заменить Excel-реестры единым реестром платежных заявок с прозрачным маршрутом, ролями, проверкой регламента, историей и уведомлениями.

## Пользовательский процесс

1. Инициатор создает черновик заявки.
2. В форме система сразу показывает, пройдет ли заявка стандартный регламент.
3. Инициатор отправляет заявку.
4. Если нарушений нет, заявка идет ФЭО на согласование.
5. Если есть нарушение, заявка получает статус **«Требует исключения»**.
6. ФЭО разрешает или отклоняет исключение.
7. После согласования заявка становится доступна кассиру для оплаты.
8. Кассир отмечает оплату.

## Исключения из регламента

Исключение требуется, если заявка нарушает правила оплаты:

- оплата сегодня после 11:00 МСК;
- дата попадает на неплатёжный день;
- дата попадает на выходной день;
- дата попадает на день выплаты зарплаты;
- категория статьи ДДС не разрешена матрицей для выбранного типа дня.

ФЭО принимает решение:

- `Разрешить исключение`;
- `Отклонить исключение`.

## Внебюджетные заявки

Если ФЭО отмечает заявку как внебюджетную, она уходит директору:

- `Утвердить вне бюджета`;
- `Отклонить вне бюджета`.

После утверждения директора заявка возвращается ФЭО на обычное согласование.

## Отложенные заявки

Согласованная, но еще не оплаченная заявка может быть отложена ФЭО. Это используется, когда платеж временно нельзя провести, но заявка не должна быть отклонена.

Маршрут:

```text
APPROVED + UNPAID -> Отложить -> SUSPENDED/Отложена -> Вернуть на согласование -> PENDING
```

## UI-принципы

- Статусы не редактируются напрямую в колонке.
- Действия выполняются через кнопки.
- Основное действие видно сразу.
- Редкие или опасные действия спрятаны в меню `...`.
- Терминология в UI бизнесовая: **исключение**, **отложена**, **вне бюджета**.
