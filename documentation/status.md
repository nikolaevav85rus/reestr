# Статус проекта

Дата актуализации: 22.04.2026.

## Текущее состояние

Система реализует единый реестр платежных заявок с ролевым workflow:

- инициатор создает и отправляет заявки;
- ФЭО согласует, отклоняет, уточняет, переносит, разрешает/отклоняет исключения и откладывает заявки;
- директор принимает решения по внебюджетным заявкам;
- кассир оплачивает согласованные заявки;
- бухгалтер ведет статус договора.

Актуальный бизнес-процесс описан в [current-workflow.md](current-workflow.md).

### 22.04.2026 — уточнение workflow вне бюджета

Сделано:

- добавлен промежуточный статус `MEMO_REQUIRED` для заявок вне бюджета;
- при выборе ФЭО `Бюджет = Нет` заявка сначала требует решения инициатора, а не уходит сразу директору;

- ФЭО/администратор могут вернуть `MEMO_REQUIRED` в черновик с новой датой, если платеж нужно перенести до решения директора;

## Текущая статусная модель

| Технический статус | UI | Ответственный |
| --- | --- | --- |
| `DRAFT` | Черновик | Инициатор |
| `PENDING_GATE` | Требует исключения | ФЭО |
| `PENDING` | На согласовании | ФЭО |
| `MEMO_REQUIRED` | Требует обоснования | Инициатор |
| `PENDING_MEMO` | Вне бюджета | Директор |
| `APPROVED` | Согласовано | Кассир / ФЭО для отложения |
| `REJECTED` | Отклонено | Конечный статус согласования |
| `CLARIFICATION` | На уточнении | Инициатор |
| `POSTPONED` | Перенесено | Инициатор |
| `SUSPENDED` | Отложена | ФЭО |

## Текущие endpoints workflow

- `POST /requests/` — создать черновик;
- `POST /requests/gate_preview` — предварительно проверить регламент;
- `POST /requests/{id}/submit` — отправить заявку;
- `POST /requests/{id}/approve_gate` — разрешить исключение;
- `POST /requests/{id}/reject_gate` — отклонить исключение;
- `POST /requests/{id}/approve` — согласовать;
- `POST /requests/{id}/reject` — отклонить;
- `POST /requests/{id}/clarify` — вернуть на уточнение;
- `POST /requests/{id}/postpone` — перенести;
- `POST /requests/{id}/memo_reason` — обосновать внебюджетную заявку;
- `POST /requests/{id}/cancel_memo` — отменить внебюджетную заявку до передачи директору;
- `POST /requests/{id}/approve_memo` — утвердить вне бюджета;
- `POST /requests/{id}/reject_memo` — отклонить вне бюджета;
- `POST /requests/{id}/suspend` — отложить;
- `POST /requests/{id}/unsuspend` — вернуть на согласование;
- `POST /requests/{id}/pay` — оплатить.

Технические имена `gate_*` и `suspend/unsuspend` оставлены для совместимости. В UI используются бизнес-термины.
