# Frontend: реестр платежных заявок

Дата актуализации: 22.04.2026.

## Стек

- React 19;
- TypeScript;
- Vite;
- Ant Design;
- Axios через `frontend/src/api/apiClient.ts`;
- Zustand для состояния авторизации;
- Playwright для UI-проверок.

## Главный экран

Основной рабочий экран — `PaymentRegistry.tsx`.

Реестр показывает платежные заявки с фильтрами, группировкой, настройкой колонок и карточкой просмотра. Клик по обычной строке открывает карточку заявки. В grouped mode строки-группы не открывают карточку, потому что внутри них несколько заявок.

## Принцип workflow UI

Колонка **«Согласование»** только показывает статус тегом. Изменение workflow выполняется через действия в колонке **«Действия»**:

- одно главное действие отображается текстовой кнопкой;
- дополнительные действия находятся в меню `...`;
- inline Select для согласования не используется.

Пользовательский термин для нестандартного маршрута: **«исключение из регламента»**.

## Действия по ролям

Инициатор:

- `Отправить`;
- `Вернуть в черновик`;
- `Редактировать`;
- `Копировать`.

ФЭО:

- `Согласовать`;
- `Отклонить`;
- `На уточнение`;
- `Перенести`;
- `Разрешить исключение`;
- `Отклонить исключение`;
- `Отложить`;
- `Вернуть на согласование`.

Директор:

- `Утвердить вне бюджета`;
- `Отклонить вне бюджета`.

Кассир:

- `Оплатить`.

Бухгалтер:

- workflow-действий нет;
- inline-поле **«Договор»** остается доступным по праву `req_set_contract`.

## Форма заявки

Форма создания/редактирования заявки содержит:

- организацию;
- ЦФО;
- контрагента;
- статью ДДС;
- назначение платежа;
- описание;
- сумму;
- дату оплаты;
- примечание;
- файл.

После выбора организации, статьи ДДС и даты оплаты форма вызывает `POST /requests/gate_preview` и показывает подсказку:

- **«Заявка разрешена»** — стандартный маршрут;
- **«Заявка требует дополнительного согласования»** — маршрут через исключение с причиной.

## Статусы в UI

- `DRAFT` — Черновик;
- `PENDING_GATE` — Требует исключения;
- `PENDING` — На согласовании;
- `PENDING_MEMO` — Вне бюджета;
- `APPROVED` — Согласовано;
- `REJECTED` — Отклонено;
- `CLARIFICATION` — На уточнении;
- `POSTPONED` — Перенесено;
- `SUSPENDED` — Отложена.

## Реестр

Поддерживается:

- фильтрация по датам, организации, ЦФО, контрагенту, назначению, категории ДДС, статье ДДС, сумме, согласованию, оплате и пометке на удаление;
- группировка по организации, категории ЦФО и категории ДДС;
- sticky horizontal scrollbar;
- настройка видимости и ширины колонок;
- открытие файла из строки и карточки;
- просмотр истории в карточке заявки.
