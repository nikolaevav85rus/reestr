# Архитектурные и системные требования (Pilot Phase)

## 1. Технологический стек (Backend)
* **Фреймворк:** FastAPI (Python 3.10+). Обеспечивает нативную асинхронность и автоматическую генерацию OpenAPI (Swagger) документации.
* **СУБД:** PostgreSQL 15+. Выбрана для обеспечения ACID-транзакций, строгой структуры данных и поддержки иерархических запросов.
* **ORM:** SQLAlchemy 2.0 (asyncio) + Alembic для управления миграциями схем БД.
* **Планировщик задач:** `APScheduler`. Выбран в качестве альтернативы Celery/Redis для стабильной работы асинхронных фоновых задач (cron) в среде Windows.

## 2. Инфраструктура и развертывание
* **ОС Сервера:** Windows Server (в локальной корпоративной сети).
* **Веб-сервер:** ASGI-сервер Uvicorn, работающий за Reverse Proxy (IIS с HttpPlatformHandler или Nginx для Windows).
* **Хранилище файлов:** Локальная файловая система сервера. В базу данных сохраняется только относительный путь (`FileStoragePath`). Файлы раздаются через `StaticFiles` во FastAPI.

## 3. Аутентификация и авторизация (Подготовка к AD)
* **Механизм:** Stateless JWT (JSON Web Tokens).
* **Интеграция с Active Directory:** * На этапе пилота используется локальная связка `ad_login` + `hashed_password` в БД.
  * Архитектура спроектирована под будущую бесшовную замену локальной проверки пароля на LDAP-bind запрос к контроллеру домена (без изменения механизма выдачи токенов).
* **Разграничение прав (RLS):** Доступ к записям фильтруется на уровне ORM (Row-Level Security) на основе ролей (INITIATOR, FEO, CASHIER, DIRECTOR, ACCOUNTING) и привязки к подразделению `direction_id`.

## 4. Ключевые алгоритмы (Бизнес-логика)
* **Конечный автомат (FSM):** Строгий контроль переходов статусов заявок (Draft -> On Review -> Approved -> Paid и т.д.).
* **Gatekeeper (Умный шлюз):** * Автоматический перенос даты платежа при подаче заявки после 11:00.
  * Валидация платежных дней в зависимости от группы компании (Пн/Ср или Вт/Чт).
  * Блокировка выплат в пятницу ("Правило Пятницы"), кроме исключений.
* **Аудит (Audit Log):** Бессрочное логирование всех изменений (Кто, Когда, Какое поле, Было, Стало).
