# Требования к системе

Дата актуализации: 22.04.2026.

## Назначение

Система ведет единый реестр платежных заявок: от создания инициатором до согласования, исключений из регламента, внебюджетных решений, оплаты и истории.

## Роли

- **ADMIN** — администрирование системы и полный доступ.
- **INITIATOR** — создание и сопровождение своих заявок.
- **FEO** — согласование, отклонение, уточнение, перенос, решение по исключениям, отложение заявок.
- **CASHIER** — оплата согласованных заявок.
- **ACCOUNTING** — проверка/ведение статуса договора.
- **DIRECTOR** — решение по внебюджетным заявкам.

## Workflow

Актуальный workflow описан в [current-workflow.md](current-workflow.md).

Основные правила:

- статус согласования в реестре отображается только тегом;
- все переходы выполняются через действия в строке;
- основное действие отображается текстовой кнопкой;
- дополнительные действия доступны через меню `...`;
- бухгалтерский статус договора остается inline-полем;
- технические поля `PENDING_GATE`, `approve_gate`, `reject_gate`, `gate_reason` не переименовываются.

## Проверка регламента оплаты

При создании и редактировании заявки инициатор должен заранее видеть, пройдет ли заявка стандартный маршрут.

После выбора организации, статьи ДДС и даты оплаты UI показывает:

- **«Заявка разрешена»**;
- **«Заявка требует дополнительного согласования»** с причиной.

Проверка учитывает:

- время после 11:00 МСК для оплаты сегодня;
- платежную группу организации;
- тип дня в платежном календаре;
- матрицу ДДС для категории выбранной статьи.

## Исключение из регламента

Если заявка нарушает регламент, она получает статус **«Требует исключения»** (`PENDING_GATE`).

ФЭО может:

- `Разрешить исключение` — заявка переходит в `PENDING`;
- `Отклонить исключение` — заявка переходит в `REJECTED`.

## Вне бюджета

Если ФЭО отмечает, что заявка вне бюджета, она переходит в `MEMO_REQUIRED`.

Инициатор сначала выбирает:

- `Обосновать` — заявка переходит в `PENDING_MEMO` и передается директору;
- `Отменить` — заявка переходит в `REJECTED`;
- `Вернуть в черновик` — заявка переходит в `DRAFT` с возможностью переноса даты.

После обоснования директор может:

- `Утвердить вне бюджета` — заявка возвращается в `PENDING`;
- `Отклонить вне бюджета` — заявка переходит в `REJECTED`.

## Отложенные заявки

ФЭО может отложить согласованную неоплаченную заявку:

```text
APPROVED + UNPAID -> Отложить -> SUSPENDED/Отложена
```

Затем ФЭО может вернуть ее на согласование с новой датой:

```text
SUSPENDED -> Вернуть на согласование -> PENDING
```

Кассир не откладывает заявки, он только оплачивает.
