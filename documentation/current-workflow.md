# Актуальный workflow заявок

Дата актуализации: 22.04.2026.

## Основная идея

Реестр работает как единый маршрут платежной заявки. Колонка **«Согласование»** только показывает текущий статус тегом. Все переходы выполняются через действия в строке: одно главное действие отображается текстовой кнопкой, остальные доступны в меню `...`.

В пользовательском интерфейсе используется бизнес-термин **«исключение из регламента»**. Технические API-поля остаются прежними: `PENDING_GATE`, `approve_gate`, `reject_gate`, `gate_reason`.

## Статусы согласования

| Технический статус | Отображение в UI | Смысл |
| --- | --- | --- |
| `DRAFT` | Черновик | Заявка создана, но не отправлена на согласование |
| `PENDING_GATE` | Требует исключения | Заявка нарушает регламент оплаты и требует решения ФЭО |
| `PENDING` | На согласовании | Заявка ожидает согласования ФЭО |
| `PENDING_MEMO` | Вне бюджета | Заявка требует решения директора по внебюджетному расходу |
| `APPROVED` | Согласовано | Заявка согласована и ожидает оплаты |
| `REJECTED` | Отклонено | Заявка отклонена |
| `CLARIFICATION` | На уточнении | ФЭО вернул заявку инициатору для уточнения |
| `POSTPONED` | Перенесено | ФЭО перенес заявку; инициатор может вернуть ее в черновик |
| `SUSPENDED` | Отложена | Согласованная неоплаченная заявка временно отложена ФЭО |

Статус оплаты хранится отдельно: `UNPAID` или `PAID`.

## Проверка регламента оплаты

Проверка выполняется при отправке заявки на согласование и доступна заранее в форме заявки через `POST /requests/gate_preview`.

Заявка проходит стандартный маршрут, если для выбранных **организации**, **статьи ДДС** и **даты оплаты** нет нарушений:

- дата оплаты не сегодня после 11:00 МСК;
- день в платежном календаре группы организации не запрещает выбранную категорию ДДС;
- если день имеет тип `NON_PAYMENT`, `HOLIDAY` или `SALARY_DAY`, категория статьи ДДС должна быть разрешена в матрице `DayTypeRule`.

Если есть нарушение, заявка получает статус **«Требует исключения»** (`PENDING_GATE`) и причину в `gate_reason`, например:

- `Заявка подана после 11:00 МСК (14:32)`;
- `Дата оплаты 2026-04-24 — день выплаты зарплаты`;
- `Дата оплаты 2026-04-26 — выходной день`;
- `Дата оплаты 2026-04-27 — неплатёжный день`.

В форме создания/редактирования после выбора организации, статьи ДДС и даты оплаты показывается подсказка:

- **«Заявка разрешена»**;
- **«Заявка требует дополнительного согласования»** с причиной.

## Роли и действия

### Инициатор

Главные действия:

- `Отправить` для `DRAFT`, `CLARIFICATION`, `POSTPONED`;
- `Вернуть в черновик` для заявок, которые нужно поправить после переноса или внебюджетного решения;
- `Редактировать` доступно для своих черновиков;
- `Копировать` доступно для создания новой заявки на основе существующей.

Инициатор не согласует заявку и не меняет статус напрямую.

### ФЭО

Для `PENDING`:

- главное действие: `Согласовать`;
- меню: `Отклонить`, `На уточнение`, `Перенести`.

Для `PENDING_GATE`:

- главное действие: `Разрешить исключение`;
- меню: `Отклонить исключение`.

Для `APPROVED` + `UNPAID`:

- `Отложить`.

Для `SUSPENDED`:

- `Вернуть на согласование` с новой датой оплаты.

### Директор

Для `PENDING_MEMO`:

- главное действие: `Утвердить вне бюджета`;
- меню: `Отклонить вне бюджета`.

### Кассир

Для `APPROVED` + `UNPAID`:

- главное действие: `Оплатить`.

Повторная оплата должна быть недоступна или заблокирована.

### Бухгалтер

Workflow-действий нет. Бухгалтер работает с inline-полем **«Договор»**:

- необработано;
- есть;
- нет.

## Основные маршруты

```text
DRAFT
  -> Отправить
  -> PENDING
  -> Согласовать
  -> APPROVED
  -> Оплатить
  -> PAID
```

```text
DRAFT
  -> Отправить
  -> PENDING_GATE
  -> Разрешить исключение
  -> PENDING
  -> Согласовать
  -> APPROVED
```

```text
DRAFT
  -> Отправить
  -> PENDING_GATE
  -> Отклонить исключение
  -> REJECTED
```

```text
PENDING
  -> На уточнение
  -> CLARIFICATION
  -> Отправить
  -> повторная проверка регламента
```

```text
PENDING
  -> Перенести
  -> POSTPONED
  -> Вернуть в черновик
  -> DRAFT
```

```text
PENDING
  -> бюджет = нет
  -> PENDING_MEMO
  -> Утвердить вне бюджета
  -> PENDING
```

```text
APPROVED + UNPAID
  -> Отложить
  -> SUSPENDED
  -> Вернуть на согласование
  -> PENDING
```

## Реестр

В реестре доступны:

- фильтры по дате, организации, ЦФО, контрагенту, назначению, категории ДДС, статье ДДС, сумме, согласованию, оплате и пометке на удаление;
- группировка по организации, категории ЦФО и категории ДДС;
- клик по обычной строке открывает карточку заявки;
- в grouped mode клик по строкам-группам не открывает карточку;
- клики по интерактивным элементам строки не открывают карточку случайно;
- настройки видимости и ширины колонок;
- sticky horizontal scrollbar.

## Документы и файлы

Разрешенные форматы вложений:

- `.pdf`;
- `.jpg`;
- `.jpeg`;
- `.png`.

Файлы открываются из строки реестра и карточки заявки. Недопустимые форматы должны отклоняться с понятным сообщением.
