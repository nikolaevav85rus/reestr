# Backend: реестр платежных заявок

Дата актуализации: 22.04.2026.

## Назначение

Backend предоставляет FastAPI API для создания, согласования, контроля и оплаты платежных заявок. Основной бизнес-процесс описан в [current-workflow.md](current-workflow.md).

## Технологии

- FastAPI;
- SQLAlchemy 2 async;
- PostgreSQL;
- Alembic;
- JWT-аутентификация;
- файловое хранилище для вложений.

## Основные сущности

- `PaymentRequest` — платежная заявка.
- `Organization` — юридическое лицо, привязанное к платежной группе.
- `PaymentGroup` — группа платежного календаря.
- `Direction` — ЦФО.
- `BudgetItem` — статья ДДС с категорией.
- `PaymentCalendar` — календарный день конкретной платежной группы.
- `DayTypeRule` — матрица разрешенных категорий ДДС для типов дней.
- `AuditLog` — история изменений.
- `Notification` — уведомления пользователей.

## Проверка регламента оплаты

Проверка вынесена в общую backend-логику и используется в двух местах:

- `POST /requests/gate_preview` — предварительная подсказка в форме заявки;
- `POST /requests/{id}/submit` — фактический переход из черновика.

Проверка учитывает:

- дату оплаты сегодня после 11:00 МСК;
- платежную группу выбранной организации;
- тип дня в `PaymentCalendar`;
- категорию выбранной статьи ДДС;
- матрицу `DayTypeRule`.

Если выбранная дата и категория ДДС разрешены для типа дня, заявка проходит стандартно. Если нет, `submit` переводит заявку в `PENDING_GATE`, а UI показывает это как **«Требует исключения»**.

Примеры причин:

- `Заявка подана после 11:00 МСК (14:32)`;
- `Дата оплаты 2026-04-24 — день выплаты зарплаты`;
- `Дата оплаты 2026-04-26 — выходной день`;
- `Дата оплаты 2026-04-27 — неплатёжный день`.

## Статусы

Approval status:

- `DRAFT` — черновик;
- `PENDING_GATE` — требует исключения из регламента;
- `PENDING` — на согласовании ФЭО;
- `PENDING_MEMO` — вне бюджета, ожидает решения директора;
- `APPROVED` — согласовано;
- `REJECTED` — отклонено;
- `CLARIFICATION` — на уточнении;
- `POSTPONED` — перенесено;
- `SUSPENDED` — отложена.

Payment status:

- `UNPAID` — не оплачено;
- `PAID` — оплачено.

## Ключевые endpoints заявок

- `POST /requests/` — создать черновик.
- `PUT /requests/{id}` — отредактировать заявку.
- `DELETE /requests/{id}` — удалить заявку.
- `POST /requests/gate_preview` — проверить, пройдет ли заявка стандартный регламент.
- `POST /requests/{id}/submit` — отправить заявку на согласование.
- `POST /requests/{id}/approve_gate` — разрешить исключение из регламента.
- `POST /requests/{id}/reject_gate` — отклонить исключение из регламента.
- `POST /requests/{id}/approve` — согласовать заявку.
- `POST /requests/{id}/reject` — отклонить заявку.
- `POST /requests/{id}/clarify` — вернуть на уточнение.
- `POST /requests/{id}/postpone` — перенести заявку.
- `POST /requests/{id}/approve_memo` — утвердить вне бюджета.
- `POST /requests/{id}/reject_memo` — отклонить вне бюджета.
- `POST /requests/{id}/suspend` — отложить согласованную неоплаченную заявку.
- `POST /requests/{id}/unsuspend` — вернуть отложенную заявку на согласование с новой датой.
- `POST /requests/{id}/pay` — отметить оплату.
- `POST /requests/{id}/upload` — загрузить файл.
- `GET /requests/{id}/file` — получить файл.
- `GET /requests/{id}/history` — получить историю заявки.

## Права

- `req_create` — создание, редактирование своих заявок, отправка.
- `req_approve` — согласование ФЭО.
- `gate_approve` — решение по исключению из регламента.
- `memo_approve` — решение директора по внебюджетному расходу.
- `req_pay` — оплата.
- `req_suspend` — отложение и возврат отложенной заявки.
- `req_set_contract` — изменение статуса договора.
- `req_view_own` / `req_view_all` — просмотр заявок с учетом RLS.

## Файлы

Разрешенные форматы:

- `.pdf`;
- `.jpg`;
- `.jpeg`;
- `.png`.

Недопустимые форматы отклоняются на backend с `400`.
