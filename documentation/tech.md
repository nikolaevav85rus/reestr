# Техническая логика

Дата актуализации: 22.04.2026.

## Статусная модель

Заявка имеет два независимых статуса:

- `approval_status` — маршрут согласования;
- `payment_status` — факт оплаты.

`approval_status`:

- `DRAFT`;
- `PENDING_GATE`;
- `PENDING`;
- `MEMO_REQUIRED`;
- `PENDING_MEMO`;
- `APPROVED`;
- `REJECTED`;
- `CLARIFICATION`;
- `POSTPONED`;
- `SUSPENDED`.

`payment_status`:

- `UNPAID`;
- `PAID`.

## Проверка регламента

Проверка регламента оплаты реализована на backend и переиспользуется:

- в `POST /requests/gate_preview`;
- в `POST /requests/{id}/submit`.

Логика:

1. Если дата оплаты сегодня и текущее время в Москве после 11:00, требуется исключение.
2. Для организации определяется платежная группа.
3. Для даты оплаты и платежной группы ищется день в `PaymentCalendar`.
4. Если тип дня не `PAYMENT`, проверяется матрица `DayTypeRule`.
5. Если категория статьи ДДС разрешена для типа дня, заявка проходит стандартно.
6. Если категория не разрешена, заявка требует исключения из регламента.

Типовые причины:

- после 11:00 МСК;
- неплатёжный день;
- выходной день;
- день выплаты зарплаты.

## Workflow endpoints

- `submit` — отправка черновика с проверкой регламента;
- `approve_gate` / `reject_gate` — решение по исключению из регламента;
- `approve` / `reject` / `clarify` / `postpone` — действия ФЭО;
- `approve_memo` / `reject_memo` — решение директора;
- `suspend` / `unsuspend` — отложение и возврат на согласование;
- `pay` — оплата.

Технические названия `gate_*` сохранены для совместимости API и базы, но в UI отображаются как действия по **исключению из регламента**.

## UI-слой действий

В реестре нет inline Select для изменения `approval_status`.

Колонка **«Согласование»** отображает только тег статуса. Все переходы выполняются через колонку **«Действия»**:

- главное действие — текстовая кнопка;
- дополнительные действия — меню `...`;
- интерактивные элементы внутри строки не открывают карточку заявки случайно.

## Аудит и уведомления

Важные переходы пишутся в историю и сопровождаются уведомлениями инициатору:

- отклонение;
- уточнение;
- перенос;
- отклонение исключения;
- внебюджетное решение;
- отложение;
- напоминания по неоплаченным заявкам.
