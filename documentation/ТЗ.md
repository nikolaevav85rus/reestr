# Техническое задание: реестр платежных заявок

Дата актуализации: 22.04.2026.

## Цель

Создать единый web-реестр платежных заявок вместо Excel-файлов и ручного движения документов.

Система должна обеспечивать:

- создание платежных заявок инициаторами;
- предварительную проверку регламента оплаты;
- согласование ФЭО;
- обработку исключений из регламента;
- обработку внебюджетных заявок директором;
- оплату кассиром;
- ведение статуса договора бухгалтером;
- историю действий и уведомления.

## Роли

| Роль | Назначение |
| --- | --- |
| ADMIN | Полный доступ и администрирование |
| INITIATOR | Создание и сопровождение своих заявок |
| FEO | Согласование и контроль заявок |
| CASHIER | Оплата согласованных заявок |
| ACCOUNTING | Статус договора |
| DIRECTOR | Решение по внебюджетным заявкам |

## Workflow

Актуальный workflow описан в [current-workflow.md](current-workflow.md).

Ключевое UI-правило: колонка **«Согласование»** показывает только статус. Все переходы выполняются через кнопки действий.

## Проверка регламента

При вводе заявки пользователь должен заранее видеть результат проверки:

- **«Заявка разрешена»**;
- **«Заявка требует дополнительного согласования»** с причиной.

Проверка учитывает:

- организацию;
- платежную группу организации;
- статью ДДС и ее категорию;
- дату оплаты;
- тип дня в платежном календаре;
- матрицу разрешенных категорий ДДС для типа дня;
- время после 11:00 МСК для оплаты сегодня.

## Основные статусы

- `DRAFT` — Черновик;
- `PENDING_GATE` — Требует исключения;
- `PENDING` — На согласовании;
- `PENDING_MEMO` — Вне бюджета;
- `APPROVED` — Согласовано;
- `REJECTED` — Отклонено;
- `CLARIFICATION` — На уточнении;
- `POSTPONED` — Перенесено;
- `SUSPENDED` — Отложена;
- `PAID` — Оплачено.

## Основные действия

Инициатор:

- `Отправить`;
- `Вернуть в черновик`;
- `Редактировать`;
- `Копировать`.

ФЭО:

- `Согласовать`;
- `Отклонить`;
- `На уточнение`;
- `Перенести`;
- `Разрешить исключение`;
- `Отклонить исключение`;
- `Отложить`;
- `Вернуть на согласование`.

Директор:

- `Утвердить вне бюджета`;
- `Отклонить вне бюджета`.

Кассир:

- `Оплатить`.

Бухгалтер:

- изменить статус договора.

## Файлы

К заявке можно приложить:

- PDF;
- JPG/JPEG;
- PNG.

Файлы Word и другие произвольные форматы должны отклоняться.

## Нефункциональные требования

- Web UI должен работать без ручного обновления страницы после действий.
- Все важные переходы должны фиксироваться в истории.
- Пользователь должен получать уведомления по событиям, требующим его внимания.
- API должен возвращать понятные ошибки для UI.
- Тестовые счета и локальные документы не должны храниться в git.
